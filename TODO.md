## 트랙 1 — 방법론 (Protocol / Methodology)

추출 방법 11개 문서가 현재 **설계 설명** 수준이다. 연구자가 문서만 보고 독립 재현하려면 **절차서** 수준이 돼야 한다.

### T1-1. 퍼실리테이터 역할 정의
각 추출 방법마다 퍼실리테이터(진행자)가 누구인지, 뭘 하는지가 없다.
- 자기 혼자 진행 가능한가? (self-administered)
- AI가 진행하는 버전과 인간이 진행하는 버전이 프로토콜 상 다른가?
- 어떤 메서드가 혼자 하기 어렵고 어떤 게 혼자 해도 되는지.

### T1-2. 중단 기준 명시
각 메서드별로 언제 세션을 끝내는지 기준이 없다.
- 래더링: 종착 가치 판단 기준, 최대 사슬 길이
- 삼항: 라운드 수, 포화(saturation) 기준
- CCRT: 에피소드 최소 수 (패턴이라고 볼 기준)
- SCT·생성 은유: 문항 수 고정인지 가변인지

### T1-3. 수치 계산 절차 (pen+paper 버전)
수식은 있는데 계산 절차가 없다.
- Dispersion score: 손으로 계산하는 단계
- Saliency: 집계 방법
- Triadic grid PCA: Python 없이 근사하려면? / Python 있으면 최소 스크립트
- Brier score (검증): 계산 예시

### T1-4. 메서드 간 의존 순서 공식화
삼항→래더링 의존 관계는 언급되지만 나머지는 불명확하다.
- 어떤 순서로 해야 데이터 품질이 높아지는가
- 순서 독립적인 메서드는 어떤 것인가
- 도메인별로 최소 메서드 조합이 뭔가

### T1-5. 에지케이스 (방법론 레벨)
- 참가자가 단답(한 단어)만 답할 때
- 세션 중간에 멈춘 경우 데이터를 쓸 수 있는가
- 한 도메인에 에피소드가 1개밖에 없을 때 CCRT를 쓸 수 있는가

## 트랙 2 — 서비스 스펙 (Service Spec)

[MVP 기획](<MVP 기획.md>)이 기반이다. 거기서 빠진 레이어들.

### T2-1. 화면 흐름 상세 (Screen Flow)
MVP 기획 §4에 화면별 설계가 있지만 **화면 간 전환 로직**이 없다.
- 각 화면의 진입 조건, 이탈 조건
- 분기 (도메인 해금 여부, 토글 상태, 검증 통과 여부)에 따른 화면 변화
- 모바일/웹 레이아웃 구분이 필요한지

### T2-2. 세션 상태 머신
추출 세션 하나가 가질 수 있는 상태와 전환이 정의돼야 한다.
- 상태: not_started / in_progress / paused / completed / abandoned
- 중간 저장 시점
- 재개 시 어느 시점부터 이어받는가
- 세션과 데이터 버전 관리

### T2-3. 데이터 모델
raw_store.yaml은 로컬 포맷이다. 서비스 레벨 데이터 모델이 필요하다.
- User, Session, Entry, ConsentToggle, VerificationRound 등 엔티티
- 친구 연결 구조 (누가 누구의 데이터를 어떤 용도로 볼 수 있는가)
- contaminated 플래그 처리

### T2-4. LLM 연동 스펙
각 활용 모드에서 LLM이 정확히 뭘 받고 뭘 뱉는지 정의돼야 한다.
- 분석 모드: 입력 포맷, 출력 포맷, 길이/스타일 제약
- 시뮬레이션 모드: 에이전트 초기화 방법, 턴 구조, 관찰자 UI 연결
- 검증 모드: 봉인 예측 생성 방법, 라인업 미끼 선택 로직
- 공통: 컨텍스트 길이 초과 시 처리 (11개 메서드 전부 주입하면 토큰이 얼마나 되나)

### T2-5. 검증 관문 구현
MVP 기획 §5에 개념은 있는데 구현 레벨이 없다.
- 봉인 예측을 어떻게 봉인하는가 (사용자가 못 보도록)
- 라인업 미끼(n-AFC)를 어떻게 구성하는가 (타인 데이터 익명화 풀)
- 원장(test ledger) 스키마
- 검증 통과 판정 기준 (수치)

### T2-6. 에지케이스 (서비스 레벨)
- 친구가 동의를 중간에 철회하면 이미 돌아간 분석 결과는 어떻게 되나
- 검증 라운드 도중 앱을 끄면
- 같은 데이터로 오염 전/후 검증이 섞이는 경우 분리 처리
- 미성년자 / 위기 신호 감지 분기

### T2-7. UI/UX 상세 (미작성 영역)
MVP 기획에 원칙은 있는데 실제 화면 컴포넌트 수준이 없다.
- 검사 진행 화면: 삼항 UI (세 항목 카드, 하나 고르기) 구체적 인터랙션
- 래더링 UI: 사슬이 깊어질수록 화면이 어떻게 변하나
- 가치 할당 UI: 합계 100 강제 인터랙션 방식
- 시뮬레이션 관찰 UI: 에이전트 대화를 어떻게 보여주나
- 검증 봉인/공개 모먼트 연출
