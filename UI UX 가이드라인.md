# 전체 UI/UX 설계 가이드라인

이 문서는 여러 추출 방법론과 모드 문서에 흩어져 있던 **UI(사용자 인터페이스) 및 UX(사용자 경험) 설계 요소들**을 개발 및 디자인 단계에서 참고하기 쉽도록 하나로 통합하여 정리한 것이다. 핵심은 심리적 저항과 피로도를 낮추면서도, 사용자가 무의식적으로 방어하거나 꾸며내는 것을 시각적, 경험적으로 차단하는 데 있다.

---

## 1. 인지적 통제와 시각적 강제

심리 검사에서 사용자가 너무 오래 생각하거나 시스템의 의도를 파악하여 '바람직한 답변'을 꾸며내는 것을 막기 위한 UI 기법들이다.

- **점진적 노출 (Progressive Disclosure):**
    - **적용:** 판단 스타일 시나리오, CCRT, 삼항 도출 등
    - 한 화면에 모든 질문을 표출하지 않는다. 예를 들어 판단 시나리오에서는 결핍 정보에 대한 입력을 완료하기 전까지 다음 단계의 질문을 화면에 렌더링하지 않는다.
- **스피드 추출 UI (Time Constraint):**
    - **적용:** 생성 은유와 문장 완성
    - 의식적인 검열을 방지하기 위해 속도감을 강제하되, 긴 문장의 타이핑에 시간 제한을 두면 스트레스를 유발하므로 카드 스와이프나 단어 탭 인터페이스에 한정하여 적용한다.
- **합산 강제 메커니즘 (Forced Sum Validation):**
    - **적용:** 맥락 속 가치 할당
    - 100점 만점을 강제하여 인플레이션을 방지한다. 암산으로 인한 피로를 줄이기 위해 "남은 코인"을 게이지 바와 슬라이더로 실시간 표시한다. 분배 총합이 정확히 100점이 되기 전까지 제출 버튼은 비활성화 상태로 유지된다.

## 2. 상호작용 방식의 다양화

반복적인 타이핑이 유발하는 피로도를 낮추기 위해 데이터 성격에 맞는 직관적인 조작감을 제공한다.

- **드래그 앤 드롭 (Drag & Drop):**
    - **적용:** 대표 인생 장면 (생애 타임라인 매핑)
    - 사건들을 연도나 날짜로 직접 선택하게 하지 않고 화면 상의 블록을 드래그하여 전후 순서를 재배치하는 캔버스 UI를 사용한다.
- **1-Tap / 슬라이더 UI (Low-Friction):**
    - **적용:** 행동 흔적 + 미니 ESM
    - 상태를 입력하는 미니 문항은 타이핑을 최소화한다. 슬라이더나 원탭 버튼을 활용하여 마찰을 극단적으로 줄인다.
- **개방형 자유 기입 (Open Elicitation):**
    - **적용:** 기본 원칙
    - 드롭다운 메뉴나 체크박스를 제공하지 않는다. 사용자가 직접 단어를 입력할 수 있는 빈 텍스트 입력 필드만 제공한다.

## 3. 온보딩 및 피로도 관리

- **단순한 진입:**
    - 이메일 인증 등 복잡한 절차 대신 랜딩 화면에서 이름을 입력하면 고유 식별 태그(`User#1234`)가 클립보드 복사 버튼과 함께 즉시 발급되게 한다.
- **게이미피케이션 프로그레스 바:**
    - 전체 진행률 대신 도메인별(일, 관계, 자기) 경험치 게이지 바를 통해 분할된 퀘스트의 달성감을 부여한다.

## 4. 예외 처리와 안전망

- **AI 안전 필터 에러의 우아한 처리:**
    - 민감한 단어 입력 시 안전 필터에 의해 에러가 발생하더라도 시스템이 중단되지 않도록 한다. 다른 표현으로 수정해 달라는 토스트 팝업을 띄우고 입력 내용을 유지한 채로 상태를 복구한다.
- **작문 데이터 유실 방지:**
    - 통신 에러로 긴 문장이 유실되는 것을 막기 위해 모든 텍스트 입력 필드에는 로컬 임시저장 기능을 적용하고 "작성 중 저장됨" 메시지를 표출한다.
- **비강제 교정 핑퐁 (Soft Nudging):**
    - 지나치게 추상적인 단어가 입력될 경우 에러를 반환하는 대신 구체화를 유도하는 넛지 메시지를 한 번 띄운다. 사용자가 수정 없이 제출하면 이를 그대로 수용한다.

## 5. 결과 및 피드백 화면

- **명시적인 원탭 피드백:**
    - 대화나 분석 산출물 하단에 피드백(정확함, 부분적임, 빗나감) 버튼을 배치하여 파인튜닝용 데이터를 자연스럽게 확보한다.
- **클립보드 중심의 데이터 공유:**
    - 데이터 내보내기 시 파일 다운로드 대신 텍스트 전체 복사 기능을 제공하여 외부 메신저로 즉시 전달할 수 있게 한다.

## 6. 심리적 안전감을 위한 디자인 시스템

사용자의 불안을 완화하고 성찰을 돕기 위한 색상 및 타이포그래피 원칙이다.

- **색상 팔레트 (Color Palette):**
    - **배경:** 완전한 흰색보다 차분한 Soft Cream 또는 오프화이트를 사용한다.
    - **포인트:** 자연과 평온을 상징하는 채도가 낮은 Muted Sage Green 계열을 사용한다.
    - **텍스트:** 눈의 피로를 덜어주는 Charcoal 계열의 어두운 회색을 사용한다.
    - **강조 요소:** 붉은색을 피하고 따뜻한 Soft Warm Orange 계열을 소극적으로 활용한다.
- **타이포그래피 (Typography):**
    - **질문/헤딩 문구:** 명조체(Serif) 계열을 사용하여 성찰적 상태 진입을 돕는다.
    - **본문 텍스트:** 둥근 고딕체(Sans-serif)를 사용하여 가독성을 확보하고 날카로운 인상을 피한다.
- **시각적 은유:**
    - 자연에서 모티브를 얻은 부드러운 UI 그래픽 요소를 선호하며 자극적인 애니메이션을 지양한다.

## 7. 접근성 가이드라인

- **명도 대비 준수:** 텍스트와 배경 간 명도 대비를 최소 4.5:1 이상으로 유지하여 시인성을 확보한다.
- **ARIA 라벨 제공:** 직관적인 UI 요소(슬라이더, 드래그 블록 등) 조작 시 스크린 리더 등 보조 기기를 지원하기 위해 ARIA 라벨을 적용한다.
- **키보드 탐색 보장:** 마우스 없이 탭 키와 방향키만으로 전체 검사 모듈을 진행할 수 있도록 포커스 순서를 관리한다.

## 8. 모바일 퍼스트 및 반응형 UX

특히 미니 ESM 모듈의 경우 일상 속 스마트폰 환경을 고려하여 모바일에 최적화한다.

- **조작 영역 최적화:** 액션 버튼을 화면 하단에 고정하여 한 손 조작 편의성을 확보한다.
- **충분한 터치 타겟:** 터치 오류를 줄이기 위해 조작 가능한 모든 요소는 최소 44x44px 이상의 터치 영역을 확보한다.
- **바텀 시트 활용:** 맥락의 단절 없이 가벼운 상호작용이 가능하도록 새로운 페이지 이동 대신 바텀 시트 UI를 적극 사용한다.
