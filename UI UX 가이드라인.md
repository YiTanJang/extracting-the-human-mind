# 전체 UI/UX 설계 가이드라인 (UI/UX Guidelines)

이 문서는 여러 추출 방법론과 모드 문서에 흩어져 있던 **UI(사용자 인터페이스) 및 UX(사용자 경험) 설계 요소들**을 개발/디자인 단계에서 참고하기 쉽도록 하나로 통합하여 정리한 것입니다. 핵심은 '심리적 저항과 피로도를 낮추면서도, 사용자가 무의식적으로 방어하거나 꾸며내는 것을 시각적/경험적으로 차단하는 것'입니다.

---

## 1. 인지적 통제와 시각적 강제 (Cognitive Control & Forcing)
심리 검사에서 사용자가 너무 오래 생각하거나, 시스템의 의도를 파악하고 '바람직한 답변'을 꾸며내는 것을 막기 위한 UI 기법들입니다.

*   **점진적 노출 (Progressive Disclosure):**
    *   **적용:** `판단 스타일 시나리오`, `CCRT`, `삼항 도출` 등
    *   한 화면에 모든 질문을 쏟아내지 않습니다. 예를 들어, 판단 시나리오에서는 "결핍 정보(뭘 더 알아야 해?)"를 입력하고 제출하기 전까지는, 다음 질문인 "타이브레이커(그래서 최종 기준은 뭐야?)" 입력란이 아예 화면에 보이지 않아야 합니다.
*   **스피드 추출 UI (Time Constraint):**
    *   **적용:** `생성 은유와 문장 완성`
    *   의식적인 검열(Impression Management)을 막기 위해 속도감을 강제하되, 긴 타이핑에 타임어택을 결합하면 작성 속도가 느린 사용자에게 극심한 스트레스와 이탈을 유발합니다. 따라서 타이머는 **'카드 스와이프'나 '단어 탭'** 인터페이스에만 국한하여 적용합니다.
*   **합산 강제 메커니즘 (Forced Sum Validation):**
    *   **적용:** `맥락 속 가치 할당`
    *   100점 만점을 강제하여 인플레이션을 막되, 암산으로 인한 수학적 뇌 과부하(Math Stress)와 짜증을 방지하기 위해 화면 상단에 **"남은 코인: 15"를 시각적 게이지 바와 슬라이더로 실시간 표시**해야 합니다. 분배 총합이 정확히 100점이 되기 전까지는 하단의 `[다음]` 버튼이 비활성화(Disabled) 상태로 묶여 있어야 합니다.

## 2. 상호작용 방식의 다양화 (Diverse Interactions)
타이핑만 반복하면 피로도가 극심해집니다. 각 데이터의 성격에 맞는 직관적인 조작감을 제공합니다.

*   **드래그 앤 드롭 (Drag & Drop):**
    *   **적용:** `대표 인생 장면` (생애 타임라인 매핑)
    *   과거 에피소드들의 순서를 정할 때, 정확한 연도나 날짜를 달력에서 고르게 하지 않습니다. 화면에 나열된 사건 블록들을 손가락(혹은 마우스)으로 드래그하여 전후 순서를 위아래로 재배치하는 캔버스 UI를 사용합니다.
*   **1-Tap / 슬라이더 UI (Low-Friction):**
    *   **적용:** `행동 흔적 + 미니 ESM`
    *   일상의 상태를 묻는 미니 문항은 타이핑을 최소화합니다. 슬라이더를 밀어 에너지를 표시하거나, "방금 일 있었어" 버튼 하나를 눌러 즉시 팝업을 띄우는 등 마찰을 극단적으로 줄입니다.
*   **개방형 자유 기입 (Open Elicitation):**
    *   **적용:** 거의 모든 모듈의 기본 원칙
    *   절대 '드롭다운 메뉴'나 '체크박스'로 기존에 셋업된 단어 목록을 주지 않습니다. 사용자가 직접 텍스트 빈칸에 타이핑하도록 빈 텍스트 인풋 필드만 제공합니다.

## 3. 온보딩 및 피로도 관리 (Fatigue Management)
*   **단순한 진입 (Tag-based Identity):**
    *   거창한 이메일 인증이나 비밀번호 설정 대신, 랜딩 화면에서 이름만 적으면 `User#1234` 형태의 고유 식별 태그가 클립보드 복사 버튼과 함께 발급되는 형태를 취합니다.
*   **게이미피케이션 프로그레스 바:**
    *   "전체 검사 15% 완료"처럼 긴 여정을 보여주면 지칩니다. `[일]`, `[관계]`, `[자기]` 등 도메인별로 분할된 퀘스트 경험치 게이지 바를 메인 허브에 띄워 짧은 달성감을 유도합니다.

## 4. 예외 처리와 안전망 (Graceful Fallback)
*   **AI 안전 필터 에러의 우아한 처리:**
    *   `두려운 자기` 등에서 부정적/민감한 단어를 입력해 LLM 안전 필터(Safety Filter)에 걸려 에러가 발생할 수 있습니다. 이때 시스템이 크래시되거나 무한 로딩에 빠지지 않도록, "AI 안전 필터에 의해 해석이 차단되었습니다. 다른 표현으로 수정해 주세요"라는 친절한 토스트 팝업과 함께 입력 내용을 잃지 않고 수정할 수 있는 상태로 되돌립니다.
*   **작문 데이터 로스 방지 (Data Loss Anxiety 해소):**
    *   AI 통신 지연이나 네트워크 에러 시 사용자가 공들여 쓴 장문이 날아갈 위험(엣지 케이스 로스)을 막기 위해, 모든 텍스트 인풋 필드에는 **로컬 임시저장(Auto-save / Debounce)** 기능을 넣고 "작성 중 저장됨" 단서를 화면에 띄워 안정감을 줘야 합니다.
*   **비강제 교정 핑퐁 (Soft Nudging):**
    *   `맥락 속 가치 할당`이나 `삼항 도출`에서 사용자가 "행복", "좋은 것" 처럼 너무 추상적인 단어를 썼을 때, 강제로 막거나 에러를 뱉지 않습니다. "구체적으로 어떤 모습인지 한 단어만 더 붙여볼까요?" 라는 말풍선을 한 번 띄워 재유도(Nudge)하되, 사용자가 무시하고 그대로 제출하면 그대로 통과시킵니다.

## 5. 결과 및 피드백 화면 (Output UI)
*   **명시적인 1-Tap 피드백:**
    *   모든 대화/시뮬레이션 모드가 끝나는 채팅창 하단이나 산출물 카드 밑에는 `[정확함]`, `[부분적임]`, `[빗나감]`의 3가지 피드백 버튼을 배치하여, 파인튜닝 라벨 데이터를 자연스럽게 수집합니다.
*   **클립보드 중심의 데이터 공유:**
    *   자신의 데이터를 내보낼 때 복잡한 JSON 파일 다운로드를 요구하지 않습니다. "데이터 복사하기" 버튼을 눌러 텍스트 채로 복사한 뒤, 카카오톡 등에 붙여넣기 하여 친구와 주고받을 수 있게 합니다.

## 6. 심리적 안전감을 위한 디자인 시스템 (Design Tokens)

심리학 및 멘탈 헬스케어 UI/UX 연구에 따르면, 고채도/형광색은 불안을 유발할 수 있으며, 사용자는 성찰적 상태에서 자극이 적은 환경을 선호합니다. 이를 반영한 기초 디자인 토큰입니다.

*   **Color Palette (저자극/자연 친화)**
    *   **Background (배경):** Soft Cream (`#F9F9F6`) 또는 차분한 오프화이트. 완전한 흰색(`#FFFFFF`)이 주는 임상적이고 차가운 느낌을 피합니다.
    *   **Primary (포인트):** Muted Sage Green (`#8BA88E`) 또는 Soft Slate Blue (`#7CA1B4`). 자연, 성장, 신뢰, 평온을 상징하는 채도가 낮은 색상을 사용합니다.
    *   **Text (텍스트):** Charcoal (`#333333`). 완전한 검은색(`#000000`)의 높은 대비가 주는 눈의 피로를 줄입니다.
    *   **Accent (행동 유도):** Soft Warm Orange (`#E59866`). 불안을 유발하는 붉은색 대신 따뜻한 계열을 소극적으로 배치합니다.
*   **Typography (문학적 성찰과 가독성의 조화)**
    *   **Headings (질문/성찰 문구):** Humanist Serif 계열 (예: *Cormorant Garamond* 또는 *DM Serif Text*). 사용자가 일기장을 쓰듯 '의도적이고 문학적인' 성찰 상태에 진입하도록 돕습니다.
    *   **Body (UI 및 본문 텍스트):** 둥근 산세리프 (예: *Poppins*, *Quicksand* 또는 *Pretendard*). 날카롭지 않고 포용력 있는 텍스트 형태(Round Terminals)로 불안감을 낮추고 가독성을 높입니다.
*   **Visual Metaphors (Biophilic Design)**
    *   과도한 애니메이션이나 기하학적 그래픽 대신, 은은한 그라데이션, 식물이 자라나는 듯한 프로그레스 바 등 자연(Nature)에서 모티브를 얻은 부드러운 UI 메타포를 지향합니다.

## 7. 접근성 (Accessibility) 가이드라인

*   **WCAG AA 대비 준수:** 텍스트와 배경 간 명도 대비(Contrast Ratio)를 최소 4.5:1 이상으로 유지하여 피로한 눈이나 저시력 환경에서도 가독성을 보장합니다.
*   **ARIA 라벨 제공:** 슬라이더, 드래그 앤 드롭 타임라인 등 시각적이고 직관적인 UI 요소일수록 스크린 리더 환경에서는 조작이 불가능해집니다. 반드시 적절한 ARIA 라벨과 대체 텍스트를 제공합니다.
*   **키보드 탐색 보장:** 마우스 없이 탭(Tab) 키와 방향키만으로 11개 추출 방법의 모든 응답과 제출이 가능하도록 포커스 트랩을 방지합니다.

## 8. 모바일 퍼스트 및 반응형 UX (Mobile/Responsive)

특히 **행동 흔적 + 미니 ESM**의 경우 일상 속에서 스마트폰으로 즉시 응답해야 하므로 모바일 최적화가 필수적입니다.

*   **엄지 영역 (Thumb Zone) 최적화:** '다음', '저장', '제출' 등 핵심 액션 버튼은 화면 하단(Sticky Bottom)에 고정하여 한 손 조작 시 무리가 없도록 설계합니다.
*   **충분한 터치 타겟 (Touch Target):** 터치 오류로 인한 짜증은 심리 검사의 오염 변인이 됩니다. 모든 상호작용 요소(버튼, 슬라이더 핸들, 드래그 블록)는 최소 **44x44px** 이상의 물리적 터치 영역을 보장합니다.
*   **바텀 시트 (Bottom Sheet) 활용:** 새로운 페이지로 이동시키는 대신, 현재 맥락을 유지한 상태에서 화면 아래에서 올라오는 바텀 시트 UI를 적극 활용하여 '가벼운 응답'이라는 인식을 줍니다.
