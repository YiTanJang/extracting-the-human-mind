# Methodology: 결정론적 삼항 도출 (Deterministic Triadic Elicitation)

> **예상 소요 시간:** 15~20분 (가장 많은 시간이 소요되는 핵심 모듈)
>
> **세 줄 요약:**
> - 사용자에게 3가지 기억(사건/사람)을 보여주고 "이 중 2개는 비슷하고 1개는 다른 이유가 뭘까요?"라고 물어본다.
> - 이 질문을 반복하며 사용자가 세상을 바라보는 무의식적인 잣대(가치관)를 끌어낸다.
> - 뻔한 대답을 막기 위해 겹치지 않고 다양한 기억을 조합하여 질문을 던진다.
>
> **설계 핵심:** 
> - **이론적 배경:** Kelly의 Repertory Grid 기법 변형
> - **주요 특징:** 음극·양극 텍스트(Verbatim)를 추출하여 구성개념 축 도출
> - **제어 방식:** LLM의 자의적 판단을 배제하고, 조합론 수학, 임베딩 거리 분석, 이중 게이트(Double Gate)를 통해 프롬프팅 및 중단 시점 통제

## 0. 이론적 전제 및 설계 원칙

- **편의의 범위 (Range of Convenience):** 구성개념은 특정 맥락 안에서만 유효하다. 이질적인 도메인(예: 가족과 업무)의 원소를 섞어 제시하는 것을 엄격히 금지하며, 세션 진입 시 단일 도메인(관계/일/자기)을 확정한다.

- **가치 중립성 (Value-Neutrality):** 원소(Element)를 수집하는 프롬프트는 감정 단어(예: 좋은, 싫은)를 철저히 배제하고, 에피소드 중심의 객관적 지표(예: 시간, 빈도, 행동)로 닻(Anchoring)을 내린다.

- **측정-개입 분리:** 시스템은 오직 "조합"과 "수학적 평정"만 제공하며, 어떠한 해석적 레이블도 선제공하지 않는다.

## 1. 원소 수집 (Element Elicitation)

사용자의 현재 상태와 경험을 8~10개의 구체적인 **'명사/동명사(Noun/Gerund)'** 배열($E_1, E_2, \dots, E_n$)로 변환하는 단계.

**[프레이밍 방지 (Anti-Framing) 필수 원칙]**
원소를 수집할 때 "왜 그 사람이 떠올랐는지", "어떤 특징이 있는지" 등 부가적인 수식어나 문맥을 묻는 스캐폴딩(마중물 텍스트)을 절대 금지한다. 사용자가 "김문철, 대화가 잘 통하는 사람"이라고 묘사하는 순간, 이어지는 삼항 도출(Triadic Elicitation) 단계에서 해당 원소를 온전하고 입체적인 인간으로 보지 못하고 '대화가 잘 통함'이라는 단일 프레임에 갇혀버리기 때문이다. 구성개념(Construct)은 원소 수집 단계가 아니라, 3개의 원소가 부딪히는 '도출 단계'에서 최초로 발생해야 한다. 따라서 입력 폼은 순수하게 대상의 이름이나 사건의 명칭만 받도록 통제한다.

### 1-1. 고정 자기-앵커 (Universal Self-Anchors)

모든 도메인에 필수적으로 포함되는 3개의 기저 원소. 추후 '자기 불일치(Self-Discrepancy)' 및 갈등 구조(Implicative Dilemma)를 수학적으로 계산하기 위한 기준점(Reference Point) 역할을 한다.

- $E_1$: **현재의 나** (해당 도메인에서의 내 모습)
- $E_2$: **이상적인 나** (내가 가장 바라는 내 모습)
- $E_3$: **두려운 나** (절대 도달하고 싶지 않은 내 모습)

### 1-2. 도메인별 결정론적 프리셋 (Domain-Specific Anchors)

선택된 도메인에 따라 아래의 문항을 순차 제시하여 구체적 대상/사건을 수집한다. (도메인당 6개 수집)

**도메인 A: 관계 (Interpersonal)**

- $E_4$: 최근 1주일 내에 가장 대화를 많이 나눈 사람 (빈도/물리적 거리)
- $E_5$: 나의 결정이나 의견에 가장 큰 영향을 미치는 사람 (권력/영향력)
- $E_6$: 함께 있을 때 침묵이 흘러도 아무렇지 않은 사람 (안전감/이완)
- $E_7$: 말이나 행동의 의도를 도무지 예측하기 어려운 사람 (인지적 거리/비예측성)
- $E_8$: 최근 나를 변명하게 만들었거나, 내 행동을 멈칫하게 했던 사람 (방어기제/갈등)
- $E_9$: 나에게 예상치 못한 작은 호의나 긍정적인 놀라움을 주었던 사람 (긍정적 위반/환대)

**도메인 B: 일/과업 (Work/Task)**

- $E_4$: 최근 한 달 동안 물리적인 시간을 가장 많이 쏟아부은 과업 (객관적 비중)
- $E_5$: 결과나 보상과 무관하게 시간 가는 줄 모르고 빠져들었던 순간 (내재적 동기/몰입)
- $E_6$: 시작하기 전부터 마음이 무거웠거나, 계속 뒤로 미루게 되었던 일 (회피 동기/마찰)
- $E_7$: 정답이 없어서 결정을 내리기까지 가장 오래 망설였던 상황 (판단 딜레마/통제감)
- $E_8$: 내 의도와 다르게 방향이 틀어졌지만, 어쩔 수 없이 따라야 했던 순간 (순응/무력감)
- $E_9$: 남들보다 유독 쉽고 편안하게, 별다른 노력 없이도 잘 해냈던 일 (자연스러운 유능감/강점)

**도메인 C: 자기/라이프 (Self/Life)**

- $E_4$: 최근 1년 내에 내 삶의 루틴을 가장 크게 바꾼 결정 (변화 수용성)
- $E_5$: 남들은 다 한다고 하지만 나는 끝까지 하지 않고 버티고 있는 것 (저항/고유 가치)
- $E_6$: 돌아보면 아무것도 아니었는데, 당시에는 세상이 끝날 것처럼 크게 다가왔던 일 (인지적 왜곡/민감도)
- $E_7$: 남들의 칭찬이나 인정 없이도 온전히 나 스스로 뿌듯했던 순간 (내적 기준)
- $E_8$: 절대 놓지 못할 것 같았는데, 막상 포기하고 나니 오히려 홀가분했던 경험 (집착과 해방)
- $E_9$: 남들에게 말하기 부끄럽지만 혼자 있을 때 유독 자주 빠져드는 상상이나 취미 (은밀한 자기/유희)

> **산출물:** 딱 9개의 원소가 담긴 배열 $E = [e_1, e_2, \dots, e_9]$

### 1-3. 살리언시(Saliency) 평정

수집된 각 원소에 대해 사용자가 직관적인 중요도/강렬함 점수($S_i$, 1~5점)를 부여한다. 이 값은 §2-3에서 알고리즘이 삼항을 선택할 때 가중치로 사용된다.

> **방법론적 근거:** 살리언시는 LLM이나 설계자가 부여하는 값이 아니라 **당사자 본인이 직접 부여하는 신호다**. 따라서 이는 "자의적 판단"이 아니라 개별 사례 연구(idiographic) 방법론상 정당한 가중치로 취급한다 — PCT(개인 구성개념 이론)가 본래 측정 대상으로 삼는 것이 바로 당사자의 주관적 의미 체계이기 때문이다.

## 2. 고밀도 아핀 평면 큐 (v=9, 24라운드) 기반 삼항 조합 알고리즘

순수 명사로 입력된 원소의 의미론적 거리를 임베딩 모델로 계산할 경우 무의미한 노이즈가 발생하므로, LLM의 개입을 완전히 배제하고 100% 수학적 확률에 기반한 **균형 불완비 블록 디자인(Balanced Incomplete Block Design, BIBD)**을 채택한다. 이는 원소 노출의 형평성을 완벽하게 보장하는 결정론적(Deterministic) 셔플링 알고리즘이다.

### 2-1. 구조적 제약 조건 (Structural Constraints)

모든 삼항(Triad)은 아래의 구조적 규칙을 반드시 통과해야 한다.

1. **상호 배타성:** 하나의 삼항은 반드시 서로 다른 3개의 원소로 구성된다.
2. **단일 풀 융합:** 과거 설계에서는 자기-앵커(현재/이상/두려운)와 일반 원소를 분리했으나, 본 설계에서는 도메인 맞춤형으로 수집된 **전체 9개 원소가 하나의 풀(Pool)로 융합되어 섞인다.** 이를 통해 사용자가 앵커와 타인을 직접 비교하게 만들어 깊은 심리적 차이를 강제 도출한다.
3. **완벽한 교차 보장 (Coverage):** 전체 24라운드 동안, 9개의 원소가 만들어내는 모든 가능한 원소 쌍(Pair)은 단 한 번도 빠짐없이 **정확히 2번씩 교차**해야 한다.

### 2-2. 24라운드 고밀도 큐 (Triad Queue)

목표로 하는 유효 축(Valid Construct)은 최대 12개지만, 사용자의 '패스(Pass)' 및 이중 게이트의 '기각(Discard)'을 대비하여, 그리고 깊은 래더링을 위한 응집력($\lambda=2$)을 확보하기 위해 총 **24라운드** 규모의 거대한 단일 큐를 생성한다.

이 큐는 $v=9, b=12, \lambda=1$ 속성의 아핀 평면(커크만 삼항계)을 베이스로 하여, 이를 2회 변형 확장한 구조를 지닌다.

**[고정 앵커 라운드 (Round 1)]**
- **수학적 우아함:** 9개 원소 아핀 평면 모델에서 원소에 인덱스(1~9)를 부여할 때, E1, E2, E3를 1, 2, 3번에 할당하면 첫 번째 평행군(Parallel Class)의 첫 번째 블록이 자연스럽게 `{E1, E2, E3}`가 됩니다. 
- **자기-불일치 필수 측정:** 따라서 세션의 가장 첫 라운드(Round 1)는 무조건 `[현재의 나(E1), 이상적인 나(E2), 두려운 나(E3)]` 조합으로 고정 제시됩니다. 이를 통해 사용자의 가장 근본적인 '자기 불일치(Self-Discrepancy)' 기준을 대칭성 훼손 없이 완벽하게 추출해 냅니다.

**[진행 로직 (Queue 순차 소진)]**
- Round 1(앵커 삼항)을 마친 후, 시스템은 생성된 나머지 삼항들을 차례대로 제시한다.
- **예비 풀(Reserve Pool) 폐지:** 과거의 Primary/Reserve 구분 없이, 사용자가 유효 축을 등록하든, 패스하든, 기각당하든 상관없이 무조건 큐의 다음 삼항으로 넘어가는 단방향 소진 방식을 따른다.
- Round 1을 제외한 나머지 삼항들의 제시 순서와 카드의 좌/중/우 배치는 난수 시드에 의해 완전히 섞여서 렌더링된다.

### 2-3. 원소 살리언시 보존 (Element Saliency Preservation)

1-3에서 당사자가 직접 부여한 원소별 중요도(Element Saliency, $S$)는 어떤 삼항을 선택할지 결정하는 데 쓰이지 않는다(선택은 2-2의 고정 배열이 100% 전담함). 대신, 도출된 구성개념(Construct)의 최종 점수를 가중(Weighting)하거나 인지 엔진 전체의 중력장을 계산할 때 쓰이기 위해 `saliency_sum`이라는 메타데이터로 합산되어 Raw Store에 보존된다.

## 3. 구성개념 추출 (Construct Elicitation)

제시된 삼항을 바탕으로 사용자의 내적 기준(Construct)을 Verbatim(원형 텍스트)으로 추출한다. 사용자의 응답을 있는 그대로 수용하며, 특정 방향으로 답변을 유도하거나 표면적인 답변을 지적하여 강제로 새로운 축을 생산하도록 압박하지 않는다. (얄팍한 응답은 24라운드 큐를 거치며 자연스럽게 고갈되어 심층 가치관으로 이어지기 때문)

### 3-1. 기저 질문 (Base Prompts) 및 인지 맹점 패스 (Delayed Pass)

1. **이질성 선택:** "이 3가지 중, 나머지 둘과 결이 다른 하나는 무엇인가?" → 선택된 원소를 $E_{odd}$로, 나머지 둘을 $E_{pair}$로 분리.
2. **인지 맹점 패스 (UI 로직):** 사용자가 3개의 원소를 보고 도저히 차이를 찾지 못할 때(인지적 맹점), **일정 시간(예: 15초) 이상 고민했을 때만 `[도저히 떠오르지 않음 (패스)]` 버튼이 지연 노출**된다. 패스 클릭 시 축을 도출하지 않고 포화 카운트도 올리지 않으며, 즉시 큐의 다음 라운드로 넘어간다.
3. **음극 도출 (Negative Pole):** " $E_{odd}$는 $E_{pair}$와 어떤 점이 근본적으로 다른가?" → 사용자 텍스트 입력 ($P_{neg}$)
4. **양극 도출 (Positive Pole):** "그렇다면, 두 묶음($E_{pair}$)이 공통적으로 가지고 있는 성질은 무엇인가?" → 사용자 텍스트 입력 ($P_{pos}$)

## 4. 행렬 평정 스윕 (Grid Rating Sweep)

새로 도출된 단일 구성개념 축($P_{neg} \leftrightarrow P_{pos}$)에 대해, 단계 1에서 수집한 전체 원소 $E$를 투사하여 수치화한다.

- **척도 설계:** 양극단을 1~7로 하는 리커트 척도 적용.
  - 1점 = $P_{neg}$ (음극에 완전히 가까움)
  - 7점 = $P_{pos}$ (양극에 완전히 가까움)
- **동작:** 사용자는 $E_1$부터 $E_n$까지 모든 원소를 이 스케일 위에 강제 배분(배치)한다.
- **수학적 갱신:** 전체 $N \times M$ Repertory Grid 행렬에 1개의 Row(크기 $N$의 벡터)가 추가된다.

## 5. 포화 판정 루프 (Saturation Gate)

사용자가 단순히 단어만 바꾼 동일한 구성개념을 중복 생산하는 것을 막고, 세션을 종료할 시점을 결정한다. 임베딩(문맥적 의미)과 평정 행렬(구조적 패턴)은 서로 다른 측면을 측정하므로, 한 도구의 한계가 조기 종료나 무한 반복을 일으키지 않도록 **이중 게이트(Double Gate) 모델**을 사용한다. 두 게이트는 서로 독립적인 증거이며, **LLM은 두 게이트가 합의에 실패했을 때만 개입한다.**

### 5-1. Gate 1: 평정 행렬 기반 상관성 분석 (구조적 게이트, Math)

- 방금 추가된 행 벡터 $V_{new}$와 기존 행 벡터들 $V_1, \dots, V_{m-1}$ 간의 피어슨 상관계수($r$)를 계산한다.
- $|r| > 0.85$인 기존 축 $V_{exist}$가 발견될 경우, 두 축이 본질적으로 동일한 구성개념을 측정하고 있다고 의심(Flag: 중복)한다.
- 이 게이트는 텍스트 의미를 전혀 거치지 않고 평정 행렬만으로 계산되는 **임베딩과 독립적인 증거이며**, 반의어를 가깝게 인식해버리는 임베딩의 극성 무시(polarity blindness) 문제를 완벽하게 보완하는 역할을 한다.

### 5-2. Gate 2: 임베딩 상대 분포 및 절대 임계치 게이트 (의미 게이트, Semantic)

- 방금 도출된 축의 텍스트($P_{neg}, P_{pos}$)를 임베딩한다.
- **임계치 판정:** 기존 구성개념들과 Cross-Encoder 기반 STS(Semantic Textual Similarity) 일치도를 측정하여, 점수가 **0.75~0.85 이상인** 축이 존재하면 의미적 '중복' 플래그를 세운다. *(※ 주의: 이 임계치는 초기 임의의 컷오프 값(Hyperparameter)이며, 향후 한국어 임베딩 모델의 특성과 테스트 결과에 따라 조정되어야 함)*
- 동시에, 절대 임계값뿐만 아니라 기존 구성개념들과의 거리 순위 분포를 통해 신규성 점수를 보조 확인한다. → **중복/신규 플래그 산출**

### 5-3. Gate 3: 의미론적 타이브레이커 (LLM)

- **호출 조건:** Gate 1과 Gate 2의 판정이 **일치하지 않을 때만** LLM을 호출한다.
  - 두 게이트 모두 "중복"으로 판정 → LLM 호출 없이 자동으로 신규 축을 기각(Discard)하고 포화 카운트 +1.
  - 두 게이트 모두 "신규"로 판정 → LLM 호출 없이 자동으로 행렬에 유지.
  - 두 게이트가 불일치 → 아래 프롬프트로 LLM을 단 한 번 호출하는 타이브레이커로만 사용하며, 호출 결과는 감사 로그와 함께 기록한다.
- **Probing Prompt (System):** 어느 게이트에서 '중복(Duplicate)' 플래그가 발생했는지에 따라 프롬프트를 동적으로 분기한다.

  - **Case A (Gate 1 수학적 중복, Gate 2 의미적 신규):**
    > "시스템 분석 결과, 사용자가 제시한 새로운 축 '$P_{neg} \leftrightarrow P_{pos}$'의 점수 패턴이 기존 축 '$P_{{old}_{neg}} \leftrightarrow P_{{old}_{pos}}$'와 거의 동일하다. 하지만 두 단어의 의미는 다르게 측정되었다. 사용자에게 두 기준이 본인에게 실질적으로 같은 상황을 의미하는지, 아니면 적용되는 뉘앙스에 차이가 있는지 묻는 질문을 생성한다."

  - **Case B (Gate 1 수학적 신규, Gate 2 의미적 중복):**
    > "시스템 분석 결과, 사용자가 제시한 새로운 축 '$P_{neg} \leftrightarrow P_{pos}$'의 의미가 기존 축 '$P_{{old}_{neg}} \leftrightarrow P_{{old}_{pos}}$'와 매우 유사하다. 하지만 대상들에 점수를 매긴 패턴은 달랐다. 사용자에게 두 기준이 겉보기엔 비슷한 단어임에도 불구하고 왜 각 대상(원소)들에게 다르게 적용되었는지, 어떤 미묘한 차이가 있는지 묻는 질문을 생성한다."

- **판단 로직:** 사용자가 "같다/비슷하다"고 응답하면 새로운 축을 기각(Discard)하고 포화(Saturation) 카운트를 +1 한다. "다르다"고 응답하면 차이를 인정하고 행렬에 유지한다.

### 5-4. 종료 조건 (Termination Rules)

아래 조건 중 하나라도 충족되면 삼항 도출 파이프라인을 즉시 종료하고 추출된 Raw Store를 확정한다.

1. **포화 도달:** 연속으로 3회의 포화(중복 기각) 판정이 발생한 경우. (사용자의 핵심 인지 공간을 모두 스캔 완료함)
2. **최대 용량 도달:** 유효 구성개념(축)이 최대 12개에 도달한 경우. 
3. **큐 소진 (피로도 한계):** 준비된 24라운드 큐를 모두 소진한 경우. (표면적인 대답만 반복하여 조기 종료의 혜택을 받지 못한 예외적 케이스)

결과적으로 파이프라인 종료 시 $N \times M$ 평정 행렬(Repertory Grid Matrix)이 도출됨. 이 행렬을 통해 각 축의 중심성(Intensity, $C$)과 자기-불일치 갈등(Dilemma, $K$)이 계산되어, LLM 개입 없이 결정론적으로 래더링 타깃 축을 선정하는 핵심 입력값이 된다.

## 6. 산출물 및 데이터 저장 (Raw Store Append)

각 축(라운드)은 음극·양극 verbatim + 평정 행 + *알고리즘 감사 로그*와 함께 append 된다. 점수 계산에 쓰인 수치는 사후 감사가 가능하도록 메타데이터로 남기되, 어떠한 해석 라벨도 붙이지 않는다.

```yaml
  - id: t0003
    ts: 2026-06-18T10:02:00+09:00
    method: triadic
    domain: work                        # 세션 진입 시 확정된 단일 도메인 (feared_self, CIT와 동일 필드명)
    round: 3
    elements_shown: ["밤샘 발표 준비", "팀장의 막판 수정", "스스로 정한 마감"]
    odd_one: "팀장의 막판 수정"
    raw_negative: "내가 통제권을 뺏기는 거라 제일 싫음"      # 음극 verbatim (P_neg)
    raw_positive: "나머지 둘은 내가 방향을 정한다는 점"        # 양극 verbatim (P_pos)
    grid_row: [6, 1, 7, 2, 5, 3, 6, 4, 7]  # 이 축에 전체 원소를 1~7로 평정한 행 벡터
    selection_audit:                    # 감사 로그: 이 삼항을 고른 근거 수치 (해석 아님)
      dispersion_score: 0.81
      element_saliency_sum: 13
```

> 중심성($C$)·갈등($K$)·construct 동일성 판정 등 *해석*은 전부 PART 2가 이 행렬을 읽어 만든다. Raw Store엔 verbatim·평정·감사 수치만 남는다.

## 7. 삼항 도출 그리드 분석 및 래더링 타깃 선발

모든 도출 세션이 끝나고 완성된 $N \times M$ 그리드를 바탕으로 핵심 지표를 계산하여 다음 모듈인 래더링의 타깃을 선발한다.

<details>
<summary>수치 계산 알고리즘 및 Python 코드</summary>

7점 척도(1–7) 그리드 행렬에서 계산한다. 행 = 구성개념(construct), 열 = 요소(element).

### 1. Dispersion Score (구성개념 변별도) — *축 수준, 사후 지표*

이 구성개념이 요소들을 얼마나 잘 변별하는가. 값이 높을수록 이 축이 사람들 사이의 차이를 잘 잡아낸다.
$$\text{dispersion}(row) = \frac{\text{std}(row)}{3.0}$$

```python
import numpy as np

def dispersion_score(row: list[float]) -> float:
    return np.std(row) / 3.0
```

### 2. Construct Deviation Sum (축 편차 합산) — *축 수준, 사후 지표*

이 구성개념에서 참가자가 얼마나 강하게 반응했는가. 중립(4점)에서 멀수록 현저하다.
$$\text{deviation}(row) = \sum_{i} |row_i - 4|$$

```python
def construct_deviation_sum(row: list[float]) -> int:
    return int(sum(abs(x - 4) for x in row))
```

### 3. 구성개념 중심성 (Construct Centrality / Bannister's Intensity)

어떤 구성개념이 이 사람의 전체 인지 구조에서 허브(Hub) 역할을 하는가.

```python
import numpy as np
from scipy import stats
from sklearn.decomposition import PCA

def construct_centrality(grid: list[list[float]]) -> tuple[list[float], float]:
    G = np.array(grid, dtype=float)
    n_constructs = G.shape[0]
    
    pca = PCA(n_components=1)
    if min(G.shape) > 1:
        pca.fit(G.T)
        pc1_ratio = float(pca.explained_variance_ratio_[0])
    else:
        pc1_ratio = 1.0

    multiplier = 1.0
    if pc1_ratio < 0.20:
        multiplier = 0.0
    elif pc1_ratio < 0.40:
        multiplier = pc1_ratio / 0.40
        
    intensities = []
    for i in range(n_constructs):
        sum_sq_r = 0.0
        for j in range(n_constructs):
            if i != j:
                r, _ = stats.pearsonr(G[i], G[j])
                if not np.isnan(r):
                    sum_sq_r += r**2
        
        normalized_intensity = (sum_sq_r / (n_constructs - 1)) if n_constructs > 1 else 0.0
        intensities.append(normalized_intensity * multiplier)
        
    return intensities, pc1_ratio
```

### 4. 딜레마 탐지 (Feixas Dilemma Criterion)

두 구성개념 사이에 강한 부적 상관이 있으면 딜레마(원소 9개 환경 기준 양측 |r|>=0.67 컷오프).

```python
from scipy import stats
import numpy as np

def detect_dilemmas(
    grid: list[list[float]],
    construct_labels: list[str],
    present_idx: int = 0,
    ideal_idx: int = 1,
    congruent_diff: float = 1.0,
    discrepant_diff: float = 3.0,
    r_threshold: float = 0.67
) -> list[dict]:
    G = np.array(grid, dtype=float)
    n = len(G)
    
    congruent_constructs = []
    discrepant_constructs = []
    
    for i in range(n):
        diff = abs(G[i][present_idx] - G[i][ideal_idx])
        if diff <= congruent_diff:
            congruent_constructs.append(i)
        elif diff >= discrepant_diff:
            discrepant_constructs.append(i)
            
    dilemmas = []
    for c in congruent_constructs:
        for d in discrepant_constructs:
            if np.std(G[c]) == 0 or np.std(G[d]) == 0:
                continue
            r, _ = stats.pearsonr(G[c], G[d])
            if abs(r) >= r_threshold:
                dilemmas.append({
                    "congruent_construct": construct_labels[c],
                    "discrepant_construct": construct_labels[d],
                    "r": round(r, 3),
                    "type": "implicative_dilemma"
                })
    return dilemmas
```

### 5. 래더링 타깃 선발 (Independent Target Selection)

```python
def get_laddering_targets(
    grid: list[list[float]],
    construct_labels: list[str],
    present_idx: int = 0,
    ideal_idx: int = 1
) -> dict:
    centralities, pc1_ratio = construct_centrality(grid)
    dilemmas = detect_dilemmas(grid, construct_labels, present_idx, ideal_idx)
    
    results = []
    for i, (row, label) in enumerate(zip(grid, construct_labels)):
        results.append({
            "construct": label,
            "deviation_sum": construct_deviation_sum(row),
            "centrality": centralities[i]
        })
    
    target_s = max(results, key=lambda x: x["deviation_sum"])["construct"]
    target_c = max(results, key=lambda x: x["centrality"])["construct"]
    
    target_k = None
    if dilemmas:
        strongest_dilemma = max(dilemmas, key=lambda x: abs(x["r"]))
        target_k = (strongest_dilemma["congruent_construct"], strongest_dilemma["discrepant_construct"])
        
    return {
        "Target_S": target_s,
        "Target_C": target_c,
        "Target_K": target_k
    }
```

### 6. 전체 파이프라인 구동 예시

```python
# 삼항 세션 결과가 들어오면
grid = [
    [6, 1, 7, 2, 5, 3, 6, 4, 7],  # construct 0: 통제감
    [2, 6, 1, 7, 3, 5, 2, 6, 1],  # construct 1: 수용
    [5, 5, 6, 2, 4, 3, 5, 2, 6],  # construct 2: 효율
]
labels = ["통제감 vs 무력감", "수용 vs 거부", "효율 vs 낭비"]

# 1. 각 구성개념 점수
for i, (row, label) in enumerate(zip(grid, labels)):
    print(f"{label}: dispersion={dispersion_score(row):.2f}, deviation_sum={construct_deviation_sum(row)}")

# 2. 중심성
centralities, pc1_ratio = construct_centrality(grid)
print(f"중심성 점수 (Bannister's Intensity) - PC1 설명력: {pc1_ratio*100:.1f}%:")
for label, c in zip(labels, centralities):
    print(f"  {label}: {c:.3f}")

# 3. 딜레마
for d in detect_dilemmas(grid, labels):
    print(f"딜레마: {d['congruent_construct']} ↔ {d['discrepant_construct']} (r={d['r']}, {d['type']})")

# 4. 래더링 타깃 선발
targets = get_laddering_targets(grid, labels)
print("래더링 타깃:", targets)
```
