# 수치 계산 절차 (Calculation Procedures)

추출 과정에서 나오는 수치들의 정확한 계산 방법. 알고리즘으로 돌아가는 것들만 여기 있다 — 래더링·CCRT 같은 질적 메서드는 해당 없음.

---

## 1. 삼항 도출 그리드 분석

7점 척도(1–7) 그리드 행렬에서 계산한다. 행 = 구성개념(construct), 열 = 요소(element).

### 1-1. Dispersion Score (구성개념 분산도)

이 구성개념이 요소들을 얼마나 잘 변별하는가. 값이 높을수록 이 축이 사람들 사이의 차이를 잘 잡아낸다.

**수식**

$$\text{dispersion}(row) = \frac{\text{std}(row)}{3.0}$$

- `std`: 모집단 표준편차 (population std, 분모 N)
- `3.0`: 1–7 척도에서 이론적 최대 std (절반이 1, 절반이 7일 때)
- 결과 범위: [0, 1]

**Python**

```python
import numpy as np

def dispersion_score(row: list[float]) -> float:
    return np.std(row) / 3.0
```

**기준**
- < 0.3: 변별력 낮음 — 이 구성개념은 요소를 잘 나누지 못함
- 0.3–0.6: 보통
- > 0.6: 높음 — 삼항 선택 우선순위 높음

---

### 1-2. Saliency Sum (현저성 합산)

이 구성개념에서 참가자가 얼마나 강하게 반응했는가. 중립(4점)에서 멀수록 현저하다.

**수식**

$$\text{saliency}(row) = \sum_{i} |row_i - 4|$$

**Python**

```python
def saliency_sum(row: list[float]) -> int:
    return int(sum(abs(x - 4) for x in row))
```

**기준** (8개 요소 기준)
- < 6: 현저성 낮음
- 6–12: 보통
- > 12: 현저성 높음 — 이 축이 참가자에게 심리적으로 중요함

---

### 1-3. 구성개념 중심성 (Construct Centrality via PCA)

어떤 구성개념이 이 사람의 전체 인지 구조를 가장 잘 대표하는가. PC1 로딩이 높은 구성개념이 중심적이다.

**절차**

1. 그리드 행렬 G (n_constructs × n_elements) 생성
2. 각 행을 행 평균으로 센터링
3. SVD 분해: G_centered = U Σ Vᵀ
4. PC1 = U의 첫 번째 열 → 각 구성개념의 절대값이 중심성

**Python**

```python
import numpy as np

def construct_centrality(grid: list[list[float]]) -> list[float]:
    """
    Returns PC1 absolute loading per construct.
    Higher = more central to the person's construct system.
    """
    G = np.array(grid, dtype=float)           # (n_constructs, n_elements)
    G_c = G - G.mean(axis=1, keepdims=True)   # center each row
    U, S, Vt = np.linalg.svd(G_c, full_matrices=False)
    return np.abs(U[:, 0]).tolist()

def pc1_variance_explained(grid: list[list[float]]) -> float:
    """PC1이 전체 분산의 얼마를 설명하는가."""
    G = np.array(grid, dtype=float)
    G_c = G - G.mean(axis=1, keepdims=True)
    _, S, _ = np.linalg.svd(G_c, full_matrices=False)
    return float(S[0]**2 / np.sum(S**2))
```

**주의**: PC1 설명 분산이 40% 미만이면 구성개념 구조가 다차원적이라는 뜻 — 중심성 해석에 주의하고 saliency 가중치를 높인다.

### 1-4. 딜레마 탐지 (Feixas Dilemma Criterion)

두 구성개념 사이에 강한 부적 상관이 있으면 딜레마 — "이 사람이 두 가치를 동시에 원하지만 그 둘이 서로 충돌한다."

**기준**: |r| > 0.35 (Feixas 연구 기반 컷오프)

**수식**: Pearson r between two construct rows

**Python**

```python
from scipy import stats

def detect_dilemmas(
    grid: list[list[float]],
    construct_labels: list[str],
    threshold: float = 0.35
) -> list[dict]:
    """
    Returns list of conflicting construct pairs.
    Negative r = implicative dilemma (wanting both but they pull apart).
    """
    G = np.array(grid, dtype=float)
    n = len(G)
    dilemmas = []
    for i in range(n):
        for j in range(i + 1, n):
            r, p = stats.pearsonr(G[i], G[j])
            if abs(r) > threshold:
                dilemmas.append({
                    "construct_a": construct_labels[i],
                    "construct_b": construct_labels[j],
                    "r": round(r, 3),
                    "type": "implicative" if r < 0 else "confirmatory"
                })
    return dilemmas
```

**해석**
- `r < -0.35`: 딜레마 — A와 B가 충돌. CCRT의 핵심 갈등과 교차 확인.
- `r > 0.35`: 확증 관계 — 둘이 같이 움직임 (이중 확인, 덜 중요)

### 1-5. 전체 강도 점수 (Intensity / Overall)

삼항 세션 전체 데이터의 총 강도. 중심성 × 현저성 합산.

**Python**

```python
def session_intensity(
    grid: list[list[float]],
    construct_labels: list[str]
) -> list[dict]:
    centrality = construct_centrality(grid)
    results = []
    for i, (row, label) in enumerate(zip(grid, construct_labels)):
        results.append({
            "construct": label,
            "dispersion": round(dispersion_score(row), 3),
            "saliency": saliency_sum(row),
            "centrality": round(centrality[i], 3),
            "composite": round(
                0.4 * dispersion_score(row) +
                0.3 * (saliency_sum(row) / (3 * len(row))) +
                0.3 * centrality[i], 3
            )
        })
    return sorted(results, key=lambda x: x["composite"], reverse=True)
```

## 2. 검증 지표 (Verification Metrics)

### 2-1. 정규화 정확도 (Normalized Accuracy)

Stanford *Generative Agent Simulations of 1,000 People* (arXiv:2411.10109)에서 차용.

**수식**

$$\text{normalized\_acc} = \frac{\text{model} - \text{floor}}{\text{ceiling} - \text{floor}}$$

- `floor`: 콜드 베이스라인 (데이터 없이 예측, 예: 최빈값 또는 무작위)
- `ceiling`: 자기복제 천장 (같은 사람이 2주 후 동일 문항에 재응답한 일치율)
- `model`: 우리 시스템의 예측 정확도

**Python**

```python
def normalized_accuracy(
    model_score: float,
    floor: float,
    ceiling: float
) -> float:
    if ceiling <= floor:
        return 0.0
    return (model_score - floor) / (ceiling - floor)
```

**해석**
- 0.0 = 콜드 베이스라인과 동일 (no value added)
- 1.0 = 자기복제 천장 도달 (인간 자기 일관성 한계까지 맞힘)
- < 0 = 베이스라인보다 나쁨 (방법 문제)

### 2-2. Brier Score (캘리브레이션)

LLM이 "이 사람은 X를 선택할 확률 70%"라고 말할 때, 그 확률 추정이 얼마나 정확한가.

**수식**

$$BS = \frac{1}{N} \sum_{i=1}^{N} (f_i - o_i)^2$$

- $f_i$: 예측 확률 ∈ [0, 1]
- $o_i$: 실제 결과 ∈ {0, 1}
- 낮을수록 좋음. 완벽 = 0, 무작위 이진 = 0.25

**Python**

```python
def brier_score(
    forecasts: list[float],
    outcomes: list[int]
) -> float:
    return float(np.mean([(f - o) ** 2 for f, o in zip(forecasts, outcomes)]))
```

### 2-3. n-AFC 우연 기준선

라인업 식별 테스트(A2)에서 k개 중 1개를 고를 때의 우연 정확도.

**수식**

$$P(\text{chance}) = \frac{1}{k}$$

**Python**

```python
def chance_baseline(k: int) -> float:
    return 1.0 / k
```

**사용 예**
- k=5 라인업: 우연 = 0.20 (20%)
- k=10: 우연 = 0.10
- 모델 정확도가 chance baseline을 유의미하게 초과해야 방법론 sanity 통과

## 3. 래더링 체인 처리

수치 계산은 없음. 알고리즘적 처리만:

```python
MAX_DEPTH = 4

def should_stop_laddering(
    chain: list[str],
    new_response: str
) -> tuple[bool, str]:
    """Returns (stop, reason)."""
    if len(chain) >= MAX_DEPTH:
        return True, "max_depth"
    # 반복 감지: 의미 유사 응답 (구현 시 embedding similarity > 0.92)
    if new_response.strip() in chain:
        return True, "repetition"
    # 종착 가치 신호 감지 (LLM이 판단)
    terminal_signals = ["그냥", "그게 삶의", "이게 제 전부", "더 이유가 없어요"]
    if any(sig in new_response for sig in terminal_signals):
        return True, "terminal_value"
    return False, ""
```

## 4. 전체 파이프라인 예시

# 삼항 세션 결과가 들어오면
grid = [

    [2, 6, 1, 7, 3, 5, 2, 6],  # construct 1: 수용
    [5, 5, 6, 2, 4, 3, 5, 2],  # construct 2: 효율
]
labels = ["통제감 vs 무력감", "수용 vs 거부", "효율 vs 낭비"]

# 1. 각 구성개념 점수
for i, (row, label) in enumerate(zip(grid, labels)):
    print(f"{label}: dispersion={dispersion_score(row):.2f}, saliency={saliency_sum(row)}")

# 2. 중심성
centralities = construct_centrality(grid)

for d in detect_dilemmas(grid, labels):
