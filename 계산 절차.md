# 수치 계산 절차 (Calculation Procedures)

추출 과정에서 나오는 수치들의 정확한 계산 방법. 알고리즘으로 돌아가는 것들만 여기 있다 — 래더링·CCRT 같은 질적 메서드는 해당 없음.

## 0. 삼항 조합 선택 (Triad Selection)

삼항 도출 세션(`삼항 도출.md §2-3`)에서 가장 이상적인 원소 3개의 조합(Triad)을 찾기 위한 기하학적 점수다. Max-Dispersion의 한계를 극복하기 위해 쌍거리(Rank) 중앙값 $\tau$ 부근에서 세 원소가 서로 비슷한 거리를 유지하는 중간 크기 정삼각형(low eccentricity)을 선호한다.

**Python**

```python
import numpy as np

def moderate_equilateral_score(
    ranks: list[float],     # 3개의 쌍거리 랭크 [D_rank(i,j), D_rank(j,k), D_rank(k,i)]
    saliencies: list[float],# 3개의 원소 살리언시 [S_i, S_j, S_k]
    median_rank: float,     # 전체 쌍거리 행렬의 중앙값(tau)
    lambda_var: float = 1.0,# 분산 페널티 가중치
    epsilon: float = 1e-5
) -> dict:
    """
    삼항 조합의 정삼각형 형상 점수를 계산한다.
    기하학적 페널티(Geometric Penalty)가 낮을수록, 살리언시가 높을수록 높은 점수를 반환한다.
    """
    r_arr = np.array(ranks)
    
    # 1. 중앙값으로부터의 이탈(Deviation) 페널티 합
    dev_penalty = np.sum(np.abs(r_arr - median_rank))
    
    # 2. 세 거리 간의 분산(Variance) 페널티 (찌그러진 삼각형 방지)
    var_penalty = np.var(r_arr)
    
    # 최종 기하학적 페널티
    geom_penalty = dev_penalty + (lambda_var * var_penalty)
    
    saliency_sum = sum(saliencies)
    score = saliency_sum / (geom_penalty + epsilon)
    
    return {
        "score": score,
        "geom_penalty": geom_penalty,
        "eccentricity": var_penalty,
        "median_rank_used": median_rank
    }
```

## 1. 삼항 도출 그리드 분석

7점 척도(1–7) 그리드 행렬에서 계산한다. 행 = 구성개념(construct), 열 = 요소(element).

### 1-1. Dispersion Score (구성개념 변별도) — *축 수준, 사후 지표*

> ⚠️ 용어 주의: 삼항 선택 알고리즘인 **"Moderate Equilateral Score(정삼각형 형상 점수, §0 참조)"**와 혼동하지 않도록 주의한다. 여기서 말하는 Dispersion은 오직 수집이 완료된 그리드 행 하나의 변별 강도(axis discriminability, post-collection)만을 의미한다.

이 구성개념이 요소들을 얼마나 잘 변별하는가. 값이 높을수록 이 축이 사람들 사이의 차이를 잘 잡아낸다.

**수식**

$$\text{dispersion}(row) = \frac{\text{std}(row)}{3.0}$$

- `std`: 모집단 표준편차 (population std, 분모 N)
- `3.0`: 1–7 척도에서 이론적 최대 std (절반이 1, 절반이 7일 때)
- 결과 범위: [0, 1]

**Python**

```python
import numpy as np

def dispersion_score(row: list[float]) -> float:
    return np.std(row) / 3.0
```

**기준**
- < 0.3: 변별력 낮음 — 이 구성개념은 요소를 잘 나누지 못함
- 0.3–0.6: 보통
- > 0.6: 높음 — 삼항 선택 우선순위 높음

### 1-2. Construct Deviation Sum (축 편차 합산) — *축 수준, 사후 지표*

> ⚠️ 용어 주의: 삼항 도출 시 사용자가 매기는 원소 수준의 중요도(Element Saliency Sum)와 혼동을 피하기 위해 **'Construct Deviation Sum'**이라는 명칭을 엄격히 분리하여 사용한다.

이 구성개념에서 참가자가 얼마나 강하게 반응했는가. 중립(4점)에서 멀수록 현저하다.

**수식**

$$\text{deviation}(row) = \sum_{i} |row_i - 4|$$

**Python**

```python
def construct_deviation_sum(row: list[float]) -> int:
    return int(sum(abs(x - 4) for x in row))
```

**기준** (9개 요소 기준)
- < 7: 편차 낮음
- 7–14: 보통
- > 14: 편차 높음 — 이 축이 참가자에게 심리적으로 중요함

### 1-3. 구성개념 중심성 (Construct Centrality / Bannister's Intensity)

어떤 구성개념이 이 사람의 전체 인지 구조에서 허브(Hub) 역할을 하는가. 다른 모든 구성개념들과의 상관계수를 제곱하여 합산한 값이 높은 구성개념이 중심적이다 (Bannister의 Intensity).

**절차**

1. 그리드 행렬 G (n_constructs × n_elements) 생성
2. 특정 행과 나머지 모든 행 간의 피어슨 상관계수(r) 계산
3. 각각의 상관계수를 제곱($r^2$)하여 모두 합산

**Python**

```python
import numpy as np
from scipy import stats
from sklearn.decomposition import PCA

def construct_centrality(grid: list[list[float]]) -> tuple[list[float], float]:
    """
    Returns Bannister's Intensity (sum of squared Pearson correlations) per construct,
    and the PC1 explained variance ratio.
    Higher Intensity = more central (hub) to the person's construct system.
    
    [PCA Fallback Logic]
    If PC1 explained variance is < 40%, the grid is fragmented, 
    so the intensity scores are penalized by (PC1 / 0.40).
    If PC1 < 20%, intensity is completely nullified (returns 0).
    """
    G = np.array(grid, dtype=float)
    n_constructs = G.shape[0]
    
    # 1. PCA 주성분 분석 (PC1 설명력 추출)
    pca = PCA(n_components=1)
    if min(G.shape) > 1:
        # 요소(element) 차원에서의 분산을 보기 위해 전치(T)
        pca.fit(G.T)
        pc1_ratio = float(pca.explained_variance_ratio_[0])
    else:
        pc1_ratio = 1.0

    # 2. 보정 계수(Multiplier) 계산
    multiplier = 1.0
    if pc1_ratio < 0.20:
        multiplier = 0.0
    elif pc1_ratio < 0.40:
        multiplier = pc1_ratio / 0.40
        
    intensities = []
    for i in range(n_constructs):
        sum_sq_r = 0.0
        for j in range(n_constructs):
            if i != j:
                r, _ = stats.pearsonr(G[i], G[j])
                if not np.isnan(r):
                    sum_sq_r += r**2
        
        # 스케일 정규화 및 보정 계수 적용
        normalized_intensity = (sum_sq_r / (n_constructs - 1)) if n_constructs > 1 else 0.0
        intensities.append(normalized_intensity * multiplier)
        
    return intensities, pc1_ratio
```

**주의**: 이 점수는 각 가치가 네트워크의 중심 노드로서 다른 가치들과 얼마나 강력하게 결속되어 있는지를 직관적으로 보여준다.

### 1-4. 딜레마 탐지 (Feixas Dilemma Criterion)

두 구성개념 사이에 강한 부적 상관이 있으면 딜레마 — "이 사람이 두 가치를 동시에 원하지만 그 둘이 서로 충돌한다."

**기준**: |r| >= 0.67 (원소 9개 기준 p<0.05 유의수준 컷오프)

> 💡 **소규모 그리드 통계 한계 해결:** Feixas의 r≥0.35 컷오프는 **임상 환경의 대규모 그리드(원소 15개 이상)**에서 도출된 기준이다. 본 시스템의 표준 그리드(원소 9개, df=7)에서 r=0.35는 통계적 노이즈에 불과하다. 문헌의 방법론(Pearson r)을 유지하되 통계적 착시를 막기 위해, 원소 9개 환경에서는 양측 α=0.05 유의수준인 **|r|≥0.67**을 임계값으로 엄격하게 상향 적용한다. (원소가 15개 이상으로 늘어날 경우에만 Feixas의 원래 기준인 0.35로 복귀한다).

**수식**: Pearson r between two construct rows

**Python**

```python
from scipy import stats
import numpy as np

def detect_dilemmas(
    grid: list[list[float]],
    construct_labels: list[str],
    present_idx: int = 0,  # '현재의 나' 원소 인덱스
    ideal_idx: int = 1,    # '이상적인 나' 원소 인덱스
    congruent_diff: float = 1.0,
    discrepant_diff: float = 3.0,
    r_threshold: float = 0.67
) -> list[dict]:
    """
    Feixas의 함축적 딜레마(Implicative Dilemma) 검출.
    - 일치 축(Congruent): 현재 자아와 이상적 자아의 점수 차이가 작음 (만족, 유지 희망)
    - 불일치 축(Discrepant): 현재 자아와 이상적 자아의 점수 차이가 큼 (불만족, 변화 희망)
    - 딜레마: 변화를 원하는 축(증상)이 유지하고 싶은 축(가치)과 강하게 상관(r >= 0.67, 원소 9개 기준)되어 있어, 
      증상을 고치려 하면 가치관도 훼손될 것이라는 무의식적 갈등 구조.
    """
    G = np.array(grid, dtype=float)
    n = len(G)
    
    congruent_constructs = []
    discrepant_constructs = []
    
    for i in range(n):
        diff = abs(G[i][present_idx] - G[i][ideal_idx])
        if diff <= congruent_diff:
            congruent_constructs.append(i)
        elif diff >= discrepant_diff:
            discrepant_constructs.append(i)
            
    dilemmas = []
    for c in congruent_constructs:
        for d in discrepant_constructs:
            # np.std가 0인 경우 경고 방지 및 r=0 처리
            if np.std(G[c]) == 0 or np.std(G[d]) == 0:
                continue
            r, _ = stats.pearsonr(G[c], G[d])
            if abs(r) >= r_threshold:
                dilemmas.append({
                    "congruent_construct": construct_labels[c],
                    "discrepant_construct": construct_labels[d],
                    "r": round(r, 3),
                    "type": "implicative_dilemma"
                })
    return dilemmas
```

**해석**
- `congruent_construct`: 사용자가 자신의 현재 모습에 만족하여 유지하고 싶어 하는 핵심 가치.
- `discrepant_construct`: 사용자가 고치고 싶어 하는 증상 혹은 문제적 패턴.
- 둘 사이의 높은 절대 상관관계(`abs(r) >= 0.75`): "증상을 고치면 내 소중한 가치도 잃게 될 거야"라는 딜레마.

### 1-5. 래더링 타깃 선발 (Independent Target Selection)

삼항 세션 결과를 바탕으로 래더링으로 넘겨줄 핵심 축(Target) 3개를 추출한다. 래더링 기획 원칙에 따라 가중 평균을 내지 않고 각 지표별 1위 축을 독립적으로 선발한다.

**Python**

```python
def get_laddering_targets(
    grid: list[list[float]],
    construct_labels: list[str],
    present_idx: int = 0,
    ideal_idx: int = 1
) -> dict:
    centralities, pc1_ratio = construct_centrality(grid)
    dilemmas = detect_dilemmas(grid, construct_labels, present_idx, ideal_idx)
    
    results = []
    for i, (row, label) in enumerate(zip(grid, construct_labels)):
        results.append({
            "construct": label,
            "deviation_sum": construct_deviation_sum(row),
            "centrality": centralities[i]
        })
    
    # 1. 편차(Deviation) 1위 축
    target_s = max(results, key=lambda x: x["deviation_sum"])["construct"]
    
    # 2. 중심성(Centrality) 1위 축 - 네트워크 허브
    target_c = max(results, key=lambda x: x["centrality"])["construct"]
    
    # 3. 갈등(Dilemma) 대상 축 - 가장 강한 역상관을 가진 딜레마
    target_k = None
    if dilemmas:
        # 절대 상관관계가 가장 큰(가장 강하게 묶인) 딜레마
        strongest_dilemma = max(dilemmas, key=lambda x: abs(x["r"]))
        target_k = (strongest_dilemma["congruent_construct"], strongest_dilemma["discrepant_construct"])
        
    return {
        "Target_S": target_s,
        "Target_C": target_c,
        "Target_K": target_k
    }
```

## 2. 자연어(NLP) 채점 및 평가 알고리즘

검증 모드에서 강제 선택(n-AFC)은 0 또는 1로 명확히 채점되지만, 자유 응답(텍스트)이나 시뮬레이션 궤적은 스칼라 값으로 환산하는 알고리즘이 필요하다.

### 2-1. 자유 응답 점 예측 정확도 (Text Prediction Score)

사용자의 텍스트 응답(실제값)과 LLM의 텍스트 예측값을 비교하여 하나의 정확도 점수 `[0, 1]`를 산출한다. 코사인 유사도(Bi-Encoder) 방식이 지닌 '영점 왜곡(Anisotropy)' 문제를 방지하기 위해, 두 문장의 논리적 일치도를 0.0부터 1.0 사이로 완벽히 정규화하여 직접 출력하는 **Cross-Encoder (STS 또는 NLI 모델)**를 단일 채점기로 사용한다.

**Python**

```python
# 필요 패키지: pip install sentence-transformers
from sentence_transformers import CrossEncoder

# 앱 시작 시 메모리에 로드 (추론 전용이므로 CPU에서도 충분히 빠름)
# 다국어(한국어 포함) STS 모델 권장
sts_model = CrossEncoder('cross-encoder/stsb-distilroberta-base')

def text_prediction_score(
    predicted_text: str,
    actual_text: str
) -> float:
    """
    Cross-Encoder를 통해 두 문장의 의미적 일치도(Semantic Textual Similarity)를 
    0.0 ~ 1.0 스칼라 값으로 직접 도출한다.
    라벨 채점과 선형 결합(alpha)을 할 필요 없이 이 점수 자체가 최종 정확도(Accuracy)가 된다.
    """
    score = sts_model.predict([predicted_text, actual_text])
    
    # STS 모델에 따라 출력이 0~1 범위를 약간 벗어날 수 있으므로 클리핑
    return max(0.0, min(1.0, float(score)))
```

### 2-2. 자기복제 천장 점수 (Test-Retest Ceiling)

정규화 정확도의 분모가 되는 천장(Ceiling) 값이다. 사용자가 시차를 두고 동일한 프로브에 답한 두 텍스트의 일치도를 평가한다.

**Python**

```python
def test_retest_ceiling_score(
    text_time1: str,
    text_time2: str
) -> float:
    """
    시간 차를 둔 두 응답의 일치도 (천장 점수). 
    동일하게 Cross-Encoder를 사용하여 두 응답의 의미적 일치도 스칼라 값을 도출한다.
    """
    score = sts_model.predict([text_time1, text_time2])
    return max(0.0, min(1.0, float(score)))
```

### 2-3. 행동 궤적 거리 (Trajectory Distance)

시뮬레이션 검증(B1)에서, 시간에 따라 전개되는 선택의 궤적(Trajectory)이 실제와 얼마나 일치하는지 계산한다.

**Python**

```python
def trajectory_distance(
    predicted_traj: list[float], # 예: 시간에 따른 예측치 배열
    actual_traj: list[float]     # 예: ESM으로 수집된 실제값 배열
) -> float:
    """
    두 궤적 간의 평균 절대 오차(MAE) 기반 유사도 [0, 1].
    반환값이 1에 가까울수록 궤적이 일치함.
    """
    if len(predicted_traj) != len(actual_traj):
        raise ValueError("궤적 길이가 일치해야 합니다.")
        
    p_arr = np.array(predicted_traj)
    a_arr = np.array(actual_traj)
    
    # 척도 범위 내에서의 MAE 계산
    mae = np.mean(np.abs(p_arr - a_arr))
    
    # 스케일에 맞춰 정규화 필요. 여기서는 척도 최대 폭이 1.0이라 가정.
    return max(0.0, 1.0 - mae)
```

## 3. 검증 지표 (Verification Metrics)

### 3-1. 정규화 정확도 (Normalized Accuracy)

Stanford *Generative Agent Simulations of 1,000 People* (arXiv:2411.10109)에서 차용.

**수식**

$$\text{Normalized Accuracy} = \frac{\text{model} - \text{floor}}{\text{ceiling} - \text{floor}}$$

- `floor`: 콜드 베이스라인 (데이터 없이 예측, 예: 최빈값 또는 무작위)
- `ceiling`: 자기복제 천장 (같은 사람이 2주 후 동일 문항에 재응답한 일치율)
- `model`: 우리 시스템의 예측 정확도

**Python**

```python
def normalized_accuracy(
    model_score: float,
    floor: float,
    ceiling: float
) -> float:
    if ceiling <= floor:
        # 천장과 바닥이 같다는 것은 본인조차 일관성이 없다는 뜻이므로 무효 처리
        return float('nan')
    return (model_score - floor) / (ceiling - floor)
```

**해석**
- 0.0 = 콜드 베이스라인과 동일 (no value added)
- 1.0 = 자기복제 천장 도달 (인간 자기 일관성 한계까지 맞힘)
- < 0 = 베이스라인보다 나쁨 (방법 문제)

### 3-2. Brier Score (캘리브레이션)

LLM이 "이 사람은 X를 선택할 확률 70%"라고 말할 때, 그 확률 추정이 얼마나 정확한가.

**수식**

$$BS = \frac{1}{N} \sum_{i=1}^{N} (f_i - o_i)^2$$

- $f_i$: 예측 확률 ∈ [0, 1]
- $o_i$: 실제 결과 ∈ {0, 1}
- 낮을수록 좋음. 완벽 = 0, 무작위 이진 = 0.25

**Python**

```python
def brier_score(
    forecasts: list[float],
    outcomes: list[int]
) -> float:
    return float(np.mean([(f - o) ** 2 for f, o in zip(forecasts, outcomes)]))
```

### 3-3. n-AFC 우연 기준선

라인업 식별 테스트(A2)에서 k개 중 1개를 고를 때의 우연 정확도.

**수식**

$$P(\text{chance}) = \frac{1}{k}$$

**Python**

```python
def chance_baseline(k: int) -> float:
    return 1.0 / k
```

**사용 예**
- k=5 라인업: 우연 = 0.20 (20%)
- k=10: 우연 = 0.10
- 모델 정확도가 chance baseline을 유의미하게 초과해야 방법론 sanity 통과

## 4. 래더링 체인 처리

수치 계산은 없음. 알고리즘적 처리만:

```python
MAX_DEPTH = 4

def should_stop_laddering(
    chain: list[str],
    new_response: str
) -> tuple[bool, str]:
    """Returns (stop, reason)."""
    if len(chain) >= MAX_DEPTH:
        return True, "max_depth"
    # 반복 감지: 의미 유사 응답 (구현 시 embedding similarity > 0.92)
    if new_response.strip() in chain:
        return True, "repetition"
    # 종착 가치 신호 감지 (LLM이 판단)
    terminal_signals = ["그냥", "그게 삶의", "이게 제 전부", "더 이유가 없어요"]
    if any(sig in new_response for sig in terminal_signals):
        return True, "terminal_value"
    return False, ""
```
